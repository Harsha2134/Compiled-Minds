Solution Overview

Decoupled Architecture Built for Performance, Recall, and System Credibility

Proposed Solution

  • Decoupled 4-Stage Architecture: Segregates cheap hard filters from expensive semantic embedding vector computations.
  • Layer 1 (Coarse Filter): Streams data in memory-efficient chunks with a bounded min-heap capped at 1,500 candidates.
  • Layer 2 (Semantic Matching): Runs offline, dynamic INT8-quantized MiniLM transformer similarity models.
  • Layer 3 (Trust Calibration): Continuous trust scoring derived from 13 metadata checks (Easy-Apply spam, farming).
  • Layer 4 (Deterministic Reasons): Non-LLM reason generator ensuring zero hallucination.

Approach Differentiators

  • OOM-Safe Design: Throughput bounds under 568 MB RAM, enabling safe processing of 100K pools on standard CPU instances.
  • Trap & Honeypot Resistant: Explicit filters for chronological date impossibilities and profile anomalies (0% honeypot rate in top 100).
  • Bypass Services Filter: Intelligently retains candidates at IT services firms who have prior product-company experience.

JD Understanding & Candidate Evaluation

Extracting Real Professional Intent Over Simple Buzzword Matching

Key JD Requirements Extracted

  • Core Skills: Embeddings, dense retrieval, vector search (Pinecone/Weaviate/FAISS/Milvus), hybrid indexing, IR metrics (NDCG, MAP, MRR).
  • Professional Experience: Ideal sweet-spot of 5–9 Years of Experience (YOE) at product-oriented tech companies.
  • Commuting Hub Proximity: Relocation or location checks matching Noida, Pune, Mumbai, Hyderabad, Delhi NCR.

Evaluation Beyond Keywords

  • Haversine Proximity Check: Maps locations to coordinates and permits a 100km commuting radius for flexible NCR hubs.
  • Career Stability Evaluation: Evaluates chronological durations, job hoppers, and penalizes flat titles lacking growth.
  • Availability Calibration: Prioritizes active candidates; down-weights profiles with no logins and low response rates.

Ranking Methodology

Robust Score Integration Framework Combining Semantic & Behavioral Heuristics

Retrieve, Score, and Rank Flow

  • Base Score: Computes keyword match score (Layer 1) to build a coarse top-1500 heap.
  • Semantic Score: Computes cosine similarity of the JD against candidate text profiles using quantized transformers.
  • Experience Fit: Multiplies a Gaussian offset decay factor based on experience deviation from the target YOE band.
  • Score Combination:
    Final_Score = Cos_Sim * YOE_Decay * Trust_Multiplier

Models & Mathematics Used

  • Model: all-MiniLM-L6-v2 loaded locally with network off, max seq length 64, and dynamic INT8 quantization.
  • Geodistance: Haversine distance formula used to identify commuter hubs.
  • Gaussian YOE Decay:
    exp(-(YOE - Target)^2 / (2 * variance))
  • Deterministic Tie-Breaker: Sorts score-tied candidates alphabetically by candidate_id.

Explainability & Data Validation

Preventing LLM Hallucinations & Filtering Malicious Activity

Zero-Hallucination Explainability

  • Deterministic Justification: Generated justifications are rule-based, pulling factual variables (exact YOE, current title, matched skills, location) directly from candidate profiles.
  • Why this works: Avoids LLM fine-tuning or prompting for reasoning, which would exceed the 5-minute CPU runtime budget and introduce hallucinatory claims.

Data Validation & Spam Filtering

  • 13 Metadata Anomalies Checked: Evaluates signals like 0% interview completion rates, connection-farming, instant bot response times, and unverified contacts.
  • Trust Modifiers: Smoothly scales the trust score down (from 1.0 to 0.01), pushing spam-bots and test-cheaters to the bottom of the rankings.
  • Honeypot Excluders: Detects impossible chronologies (e.g. signup dates in the future) and excludes them instantly.

System Architecture

Decoupled Modules Designed for Offline Sandboxed Environments

Core Modules

  • ranking_pipeline.py: The single core engine handling coarse filters, transformer vector representations, behavioral calibration, and reasoning.
  • download_model.py: Automates fetching of offline weights during environment setup.
  • check_pipeline_health.py: Complete diagnostic suite running checks on model files, cache files, and performance throughput.

Reproduction & Hosting

  • Local Model Weights: Pushed in local_model/ to ensure the container can run with network disabled.
  • Vector Caching: Employs .embeddings_cache.pkl to cache vector math, facilitating rapid pipeline execution.
  • Hosted Sandbox App: Streamlit UI hosted on Streamlit Cloud allowing developers and organizers to test samples instantly.
